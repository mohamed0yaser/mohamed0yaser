Lesson 1: Provider (Advanced Usage)

Provider is the most common and official solution for state management in Flutter. It allows dependency injection + reactive UI updates.

Example: Counter with Provider

class Counter with ChangeNotifier {
  int value = 0;

  void increment() {
    value++;
    notifyListeners(); // UI will update
  }
}

// main.dart
void main() {
  runApp(
    ChangeNotifierProvider(
      create: (_) => Counter(),
      child: MyApp(),
    ),
  );
}

class MyApp extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    final counter = Provider.of(context);
    return MaterialApp(
      home: Scaffold(
        body: Center(
          child: Text("Value: ${counter.value}"),
        ),
        floatingActionButton: FloatingActionButton(
          onPressed: counter.increment,
          child: Icon(Icons.add),
        ),
      ),
    );
  }
}

Best practices:

  • Use ChangeNotifierProvider for mutable state.
  • Wrap only the widget tree that needs the provider.
  • Avoid large ChangeNotifiers â†’ split into multiple classes.

Lesson 2: Riverpod

Riverpod is a modern and improved state management library that fixes limitations of Provider. It's compile-safe, testable, and flexible.


// Define a provider
final counterProvider = StateProvider((ref) => 0);

// Use in UI
Consumer(
  builder: (context, ref, _) {
    final counter = ref.watch(counterProvider);
    return Text("Value: $counter");
  },
);

// Increment
ref.read(counterProvider.notifier).state++;

Lesson 3: Bloc

Bloc (Business Logic Component) uses events and states for a predictable architecture. 


class CounterEvent {}
class Increment extends CounterEvent {}

class CounterBloc extends Bloc {

    on((event, emit) => emit(state + 1));
  }
}

// Usage
BlocProvider(
  create: (_) => CounterBloc(),
  child: BlocBuilder(
    builder: (context, count) => Text("$count"),
  ),
);

Lesson 4: Redux

Redux uses a global store + reducers to manage state.


class IncrementAction {}

int counterReducer(int state, dynamic action) {
  if (action is IncrementAction) return state + 1;
  return state;
}

final store = Store(counterReducer, initialState: 0);
